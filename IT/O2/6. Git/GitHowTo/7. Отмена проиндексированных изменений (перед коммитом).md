
#### Изменим файл и проиндексируем изменения

Внесем изменение в файл и проиндексируем это изменение.

```
git add <файл>
```

Проверим состояние

```
git status
```

Состояние показывает, что изменение было проиндексировано и готово к коммиту.

#### Восстановление индекса

Команда `restore` с флагом `--staged` очищает индекс.

```
git restore --staged <файл>
```

Команда `restore` с опцией `--staged` не изменяет фактические файлы в рабочем каталоге. Поэтому файл по-прежнему содержит ненужные изменения. 
	Однако следует быть осторожным, поскольку команда `restore` без флага `--staged` также сбросит изменения в рабочем каталоге.

#### Восстановление рабочего каталога

Восстановим наш файл до состояния последнего коммита.
```
git restore <файл>
git status
```
