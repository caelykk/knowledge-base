#### **<font color="#c3d69b">Примечание 1</font>.** 
Имена классов по стандарту именования PEP 8 должны соответствовать соглашению UpperCamelCase, однако​ встроенные классы ``(`int, float, str, list, tuple, dict` и др.)`` этому правилу не следуют.

#### **​<font color="#c3d69b">Примечание 4</font>.** 
Аналогично функциям, классы поддерживают строку документации, которая доступна через атрибут `__doc__`.

Приведенный ниже код:
```python
class Cat:
    """Класс, описывающий кошку"""
    night_vision = True
    paws_count = 4


print(Cat.__doc__)
```
выводит:
```python
Класс, описывающий кошку
```
#### **<font color="#c3d69b">Примечание 5</font>.** 
Тип любого объекта можно получить через атрибут `__class__`.

Приведенный ниже код:
```python
class Cat:
    pass


cat = Cat()

print(type(cat))
print(cat.__class__)
print(type(cat) == cat.__class__)
```
выводит:
```python
<class '__main__.Cat'> 
<class '__main__.Cat'> 
True
```
#### **<font color="#c3d69b">Примечание 6</font>.** 
Атрибут класса можно изменить только через сам класс.

Приведенный ниже код:
```python
class Cat:
    night_vision = True


cat1 = Cat()
cat2 = Cat()

Cat.night_vision = False

print(cat1.night_vision)
print(cat2.night_vision)
```
выводит:
```python
False
False
```

При попытке изменения атрибута класса через его экземпляр мы лишь добавляем этому экземпляру атрибут с аналогичным именем.

Приведенный ниже код:
```python
class Cat:
    night_vision = True


cat1 = Cat()
cat2 = Cat()

cat1.night_vision = False

print(cat1.night_vision, cat1.__dict__)
print(cat2.night_vision, cat2.__dict__)
```
выводит:
```python
False {'night_vision': False} 
True {}
```

Также в примере выше можно заметить, что поиск атрибута происходит сначала в словаре атрибутов экземпляра (`cat1.__dict__, cat2.__dict__`), а затем в словаре атрибутов класса (`Cat.__dict__`).
