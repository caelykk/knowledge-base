## Почему люди используют Python?

- <font color="#b2a2c7">Качество программного обеспечения</font>
	Код Python по замыслу должен быть читабельным, а потому многократно используемым и сопровождаемым — в гораздо большей степени, чем традиционные языки написания сценариев. Согласованность кода Python облегчает его понимание, даже если он написан не нами. Вдобавок Python располагает развитой поддержкой механизмов мно­гократного применения ПО, таких как объектно-ориентированное и функцио­нальное программирование.
- <font color="#b2a2c7">Продуктивность труда разработчиков</font>
	Код Python обычно занимает от 1/3 до 1/5 части размера эквивалентного кода на C++ или Java. Следовательно приходится меньше набирать на клавиатуре, меньше отлаживать и меньше в последствии сопровождать. Кроме того, программы запускаются немедленно, без длительных шагов компиляции и связывания.
- <font color="#b2a2c7">Переносимость программ</font>
	Большинство программ Python функционирует без изменений на всех основных компьютерных платформах.
- <font color="#b2a2c7">Поддерживающие библиотеки</font>
	Вместе с Python поставляется **стандартная библиотека** - большая коллекция предварительно собранной и переносимой функциональности. Стандартная библиотека поддерживает множество решений программных задач прикладного уровня. Вдобавок Python можно расширять сторонними библиотеками и библиотеками собственной разработки.
- <font color="#b2a2c7">Интеграция компонентов</font>
	Можно легко взаимодействовать с другими частями приложения, используя различные механизмы интеграции. Такая интеграция позволяет применять Python в качестве инструмента для `настройки и расширения` продуктов. Из кода Python можно обращаться к библиотекам C и C++, его можно вызывать из программ C и C++, интегрировать с компонентами Java и .NET и т.д.
