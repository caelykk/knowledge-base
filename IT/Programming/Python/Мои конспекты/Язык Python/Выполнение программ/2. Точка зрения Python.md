## Точка зрения Python

Современные стандартные реализации Python `компилируют` (т.е. транслируют) операторы исходного кода в промежуточный формат, известный как <font color="#d99694">байт-код</font>, и затем направляет его так называемой “виртуальной машине”.

## Компиляция в байт-код

После запуска программы Python сна­чала компилирует ваш `исходный код` (операторы внутри файла) в формат, известный как `байт-код`. Компиляция является просто шагом трансляции, а `байт-код` — <font color="#b7dde8">это низко­ уровневое и независимое от платформы представление исходного кода.</font>

Компиляция почти полностью скрыта от нас. Если процесс Python имеет доступ по записи на ва­шем компьютере, тогда он будет сохранять программы в файлах с расширением `.рус` (означает *скомпилированный* (<font color="#d7e3bc">compiled</font>) исходный файл `.ру`). До выхода версии Python 3.2 эти файлы появлялись на компьютере после запуска нескольких программ там, где находились соответствующие файлы исходного кода, т.е. в *тех же самых* ката­ логах.
