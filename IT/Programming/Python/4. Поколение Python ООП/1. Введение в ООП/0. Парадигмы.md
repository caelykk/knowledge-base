## Парадигмы
В программировании существует две парадигмы (**императивное** и **декларативное**).
`Процедурное` и `объектно-ориентированное` - это парадигмы именно императивного стиля программирования.

**Императивное программирование (как нужно сделать)** — последовательность команд, которые должны быть выполнены компьютером для достижения определенного результата.

Включает:

1. **Процедурное программирование**: в котором программа состоит из набора `процедур`, каждая из которых выполняет определенные операции.
    
2. **Объектно-ориентированное программирование**: в котором программа организована в виде `объектов`, которые имеют `свойства` и `методы` для выполнения операций.
    

**Декларативное программирование** с другой стороны, описывает, *что должно быть сделано*, а не как это сделать. 

Включает:

1. **Функциональное программирование** - программа описывается как `набор функций`, которые принимают входные данные и возвращают результаты. 
    
2. **SQL** используется для описания запросов к базе данных и манипуляции данными.
    
3. **Логическое программирование**: в котором программа состоит из логических утверждений, а не последовательности команд.
    

Плюс к этому есть **мультипарадигмальность** (англ. multiparadigmality) — это свойство программирования или языка программирования, который поддерживает несколько парадигм программирования. Например, некоторые языки программирования, такие как C++ и Python, поддерживают несколько парадигм программирования, таких как процедурное программирование, объектно-ориентированное программирование, функциональное программирование и т.д.
