
`cat` — команда для чтения файлов

`cat` ожидает **аргументы** — пути до файлов, которые надо читать.

```
cat .profile

# ~/.profile: executed by the command interpreter for login shells.

# if running bash
if [ -n "$BASH_VERSION" ]; then
    # include .bashrc if it exists
    if [ -f "$HOME/.bashrc" ]; then
    . "$HOME/.bashrc"
    fi
fi
```

`head` — показывает первые 10 строк.
`tail` — последние 10 строк файла.

Тоже принимают на вход путь до файла.

`head -n` | `tail -n`  — `-n` задаёт желаемое количество строк.


### Логи
**логи** (path: `/var/log`) — это текстовые файлы, в которые заносится информация о происходящих процессах в определенной системе.

**syslog** — Основной файл логов Linux

В WSL `syslog` не включен по умолчанию. Чтобы включить его, сперва установите его как:

```
sudo add-apt-repository ppa:adiscon/v8-stable
sudo apt-get install rsyslog
```

А затем запустите, используя команду `sudo service rsyslog start`.

Представим, что нам нужно изучить события, которые происходят часто. Знакомые команды в этой ситуации не помогут:
- Пользоваться командой `cat` неудобно, потому что `syslog` слишком большой
- Команда `tail` тоже не подойдет, потому что при высокой скорости записи логов мы рискуем пропустить часть события.

В этой ситуации поможет `tail`, запущенный в особом режиме — с **флагом** `-f`.

Команда `tail -f path/to/file` не просто выводит последние строчки файла, но и ждет появления новых. Как только файл дописывается, `tail` сразу выводит на экран добавленные строки. Это команда, которая **захватывает управление**. То есть, после запуска она не заканчивает выполнение сразу, а продолжает работать, ожидая новые данные в файле, который выводится:

### Пейджеры

**Пейджеры** — Особая категория программ для просмотра содержимого файла.
Пейджер похож на текстовый редактор, открытый только в режиме чтения.

**less** - Самый распространенный пейджер.

```
# Снова придется воспользоваться sudo
sudo less syslog
# Здесь много вывода
```

Пейджер `less` открывает файл и остается в этом режиме. Он позволяет производить поиск и перемещаться по файлу вперед и назад.
Пейджер не пытается загрузить в память весь файл до его отображения. Он грузит только ту часть, которая помещается на экран и при перемещении подгружает остальное.

Основные команды для перемещения по тексту:
- Выход — `q`
- Вперед на страницу — `f`
- Назад на страницу — `b`
- Поиск введенного текста — вводим `/`, печатаем текст и нажимаем `Enter`
- Переход к следующему совпадению — `n`
- Переход к предыдущему совпадению — `N`

Когда мы запускали `man`, то перед нами открывался `less` с загруженным туда контентом.
