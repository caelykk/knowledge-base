
### URL

**URL (Uniform Resource Locator)** — это уникальный адрес ресурса в сети.

Типичный URL состоит из нескольких частей, каждая из которых отвечает за свою задачу.

```
https://www.example.com:443/path/to/resource?query=value#fragment
```

### Протокол (Scheme)

**HTTP**/**HTTPS** — это стандартные протоколы, определяющий, как будет происходить взаимодействие между клиентом и сервером.

В URL всегда указывается протокол в начале строки, за ним ставится `://`.
- `https://` — HTTP с шифрованием (по умолчанию порт 443).
- `http://` — HTTP без шифрования (по умолчанию порт 80).
- `ftp://` — FTP для обмена файлами.
- `mailto:` — ссылка для отправки email.

### Доменное имя (Host)

**Доменное имя** — это уникальный адрес сайта в интернете. Доменное имя связывает сервер с понятным пользователю именем, а не с числовым IP-адресом.

Система доменов построена иерархически, начиная с корня.
##### Поддомен

Часто доменное имя включает в себя поддомен (но может отсутствовать).

**Поддомен** — это дополнительный уровень в структуре домена, используемый для выделения частей сайта, например: `blog.example.com`.

`www` — это поддомен, его можно воспринимать как домен внутри домена

##### Домен верхнего уровня

**Домен верхнего уровня (TLD — Top-Level Domain)** — это последняя часть доменного имени, которая идет после последней точки в адресе. TLD помогает классифицировать домены по их принадлежности или географическому расположению.

Примеры:

- **Общие домены** (gTLD): `.com`, `.org`, `.net`.
- **Страны/географические зоны** (ccTLD): `.ru`, `.us`, `.de` (для России — `.ru`, США — `.us`).
- **Специальные домены** (sTLD): `.gov`, `.edu`, `.int` — эти домены часто зарезервированы для определённых типов организаций.


### Порт (PORT)

**Порт** указывает, через какую точку входа на сервер должен обращаться клиент.

Если порт стандартный (443 — для HTTPS, 80 — для HTTP), то его можно не указывать, браузер сам его подставит. Но если порт отличается, то его нужно прописывать явно.


### Путь (path)

**Путь** указывает на конкретный ресурс на сервере. Он определяет, какие именно данные или страницу нужно вернуть. Путь начинается с первого `/` и может содержать дополнительные папки и файлы. 

```
/path/to/resource
```

Если нужна домашняя страница, то большинство серверов позволяет не указывать путь.

### Параметры запроса (Query)

**Параметры запросов** — это пары «ключ-значение», которые передаются серверу, на сервер для фильтрации или уточнения запроса.
- Начинаются с символа `?` 
- Разделяются амперсандом `&`.

```
?query=value
?search=apple&category=fruits&sort=price
```

### Фрагмент (Fragment)

**Фрагмент** (или якорь) указывает на определённую часть документа, которая будет отображена пользователю.

- Начинается с `#`
- обрабатывается на стороне клиента
- не отправляется на сервер

```
#fragment
```

Фрагмент полезен для перехода на определённую часть страницы, например, на секцию с комментариями или на конкретный раздел документа.

### Пример полного URL


```
https://www.example.com:443/products?category=electronics&id=12345#reviews
```

- **Протокол:** `https://` — используем защищённое соединение.
- **Домен:** `www.example.com` — основной домен, поддомен `www`.
- **Порт:** `443` — стандартный порт для HTTPS.
- **Путь:** `/products` — мы обращаемся к разделу «продукты».
- **Параметры запроса:** `category=electronics&id=12345` — параметры фильтрации по категории и идентификатору продукта.
- **Фрагмент:** `#reviews` — переходим к разделу с отзывами.

### Роль URL в запросах

URL выполняет несколько ключевых задач:
1. Он сообщает браузеру, **к какому серверу** нужно отправить запрос.
2. Он указывает на **конкретный ресурс** на этом сервере.
3. Он может передавать **дополнительные параметры**, которые помогают серверу обработать запрос.
